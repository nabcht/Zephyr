## Development Setup

1. Fork and clone the repository.
2. Ensure you have Python 3.10+ installed.
3. Set up the virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Or venv\Scripts\activate on Windows
   pip install -r requirements.txt
   ```

## Project Architecture

μZephyr relies on a mission-style multi-agent orchestration pattern:
- **Supervisor**: Plans the mission and coordinates sub-agents.
- **Researcher**: Gathers information from local DB, vector stores, or web.
- **Coder**: Implements logic and leverages the Docker sandbox.
- **Reviewer**: Evaluates outcomes and validates sandbox tests.

## Adding New Skills

Skills are modular and loaded dynamically from the `skills/` directory.

To create a new skill:
1. Create a directory: `skills/your-skill-name/`.
2. Provide a main python script with a clearly defined entry point function.
3. Ensure you include detailed docstrings! The Orchestrator uses your docstrings to understand when and how to deploy the skill.

## Testing & Sandboxing

Any code generated by the Coder agent should be run within the **Docker-backed sandbox** if possible. When developing core features, make sure sandbox boundaries are respected and tests pass in an isolated environment.

## Pull Request Process

1. Create a feature branch (`git checkout -b feature/my-new-skill`).
2. Commit your changes with descriptive messages.
3. Push to your branch and open a Pull Request.
4. Ensure all CI/CD checks (if any) pass and await review.
