# Zephyr Documentation Index

The main `README.md` stays at the repository root as the quick-start entry point. The rest of the project documentation now lives in this `Docs/` folder.

## Product And Operator Docs

- `Features.md` - current user-facing feature inventory.
- `docs.md` - high-level feature and architecture overview.
- `glossary.md` - shared vocabulary for runtime, UI, and AI terminology.
- `DASHBOARD.md` - operator map for the current React control room.
- `API_DOCS.md` - FastAPI bridge and endpoint reference.
- `DESIGN.md` - high-level design and product direction notes.

## Governance And Contribution Docs

- `CONTRIBUTING.md` - contribution workflow and development expectations.
- `PRIVACY.md` - privacy posture and data-handling summary.
- `SECURITY.md` - security guidance for local-first operation.
- `TERMS.md` - repository terms of service.

## Status And Internal Notes

- `HYBRID_MIGRATION_STATUS.md` - migration and parity record.
- `MCP_features_status.md` - MCP integration progress and follow-up notes.
- `README_FOR_AI.md` - current AI contributor brief for the existing hybrid codebase.

## Suggested Reading Order

1. Start with the root `README.md`.
2. Read `Features.md` for the current product surface.
3. Use `docs.md` for architecture and `glossary.md` for terminology.
4. Open `DASHBOARD.md` and `API_DOCS.md` when you need operator or backend details.
5. Use the governance docs before changing distribution, privacy, or contribution workflows.
6. Open the status docs only when you need migration history or MCP-specific operational context.
